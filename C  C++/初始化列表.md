### 注意
- 成员初始化顺序，依赖于成员的声明顺序
- 初始化列表先于构造函数的函数体执行

### 类中const成员
- 类中const成员会被分配空间的，与当前环境相关
- 类中const成员本质是只读变量
- 类中const成员初始化必须在初始化列表中初始化指定值

===>>编译器无法直接得到const成员初始值，无法进入符号表成为真正意义上的常量

```cpp
#include <stdio.h>

class Value
{
private:
    int mi;
public:
    Value(int i)
    {
        printf("i = %d\n", i);
        mi = i;
    }
    int getI()
    {
        return mi;
    }
};

class Test
{
private:
	const int ci;
    Value m2;
    Value m3;
    Value m1;
public:
    Test() : m1(1), m2(2), m3(3),ci(100)//初始化对象
    {
       // printf("Test::Test()\n");
		printf("Test():ci = %d\n", ci);
    }
	int GetCi()
	{
		return ci;
	}
	int SetCi(int v)
	{
		int *p = const_cast<int*>(&ci);
		*p = v;
		return 0;
	}
};


int main()
{
    Test t;
	
	printf("ci = %d\n", t.GetCi());
	
	t.SetCi(10);
	
	printf("ci = %d\n", t.GetCi());
    
    return 0;
}

```
### 初始化与赋值
- 初始化：对正创建的对象进行初值设置
- 赋值：对已经存在的对象进行值设置

int i=0；//初始化
//i = 0;//赋值

### 小结
- 类可以使用初始化列表对成员进行初始化
- 初始化列表必须在构造函数体的前面执行
- 类总可以用const修饰成员变量，为只读变量
- const成员变量必须用初始化列表初始化
